the brief

"What's the temperature, right now, at the most extreme places on earth?" Lut Desert. Vostok Station. Death Valley. Oymyakon. Places most people will never go and most reporting never reaches. The page surfaces them as a moodpiece — not a weather forecast. The job is to make the numbers feel as extreme as they are.

the decision that mattered: dropping the live feeds

The original concept paired each extreme with the nearest watchable live feed — a YouTube Live stream or Windy webcam from the closest populated place. Three things killed it:

  • YouTube Live URLs die. Streams go offline, channels get rate-limited, embeds get age-gated.
  • Windy needs a key + has TOU constraints on embedding outside their ecosystem.
  • Honesty. The placeholder feed handles I'd designed with were illustrative — none of them existed. Shipping illustrative handles in production-looking chrome is the opposite of receipts.
The cleanest version of an idea is usually the one you arrive at by removing things, not adding.

architecture

Three parts. None of them complicated.

1 · the cron — open-meteo as the data source

Open-Meteo's ERA5 forecast endpoint is free, has no auth, and gives current temperature plus the last 24 hours of hourly readings for any lat/lon. A Node script walks 14 canonical extremes (one hot + one cold per continent) and pulls live numbers. Fourteen API calls, ~1.4 seconds.

const u = new URL('https://api.open-meteo.com/v1/forecast');
u.searchParams.set('latitude',  lat);
u.searchParams.set('longitude', lon);
u.searchParams.set('current',   'temperature_2m');
u.searchParams.set('hourly',    'temperature_2m');
u.searchParams.set('past_hours', '24');

2 · the schedule — github actions, every 15 minutes

on:
  schedule:
    - cron: '*/15 * * * *'
  workflow_dispatch: {}

Commits the JSON if it changed, pushes. Vercel auto-redeploys. Zero dollars, zero per-visitor API calls — page loads a static JSON file like any other static asset, so it can't get rate-limited even if it goes viral.

3 · the page — vanilla js, mapbox gl, no framework

Standalone hot-cold.html + .css + .js. No build step, no React, no bundler. On load, fetches the cached JSON; on failure falls back to an embedded fixture so the UI never breaks. Total wire weight under a hundred kilobytes excluding Mapbox tiles.

design system

Cobalt — the second of the two systems on raghavahuja.com. Type pairing is Fraunces (italic display) × Geist (body) × JetBrains Mono (chrome). Accent #5b8cff on paper #0c0c0e. Hot uses ember #c8553d; cold reuses cobalt as its color. One warm accent in the whole palette.

what's next

  • Extensible to other extremes. Top tabs already render wind / rain / uv as "soon." Adding them is render variation, not new infra.
  • Bigger candidate registry. Twenty per continent would catch true daily winners.
  • OG image. The split red/blue is unusually well-suited to a static OG.
