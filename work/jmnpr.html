enterprise deals, by design

JMNPR Labs is a one-person studio I started in my evenings, after thirteen years of shipping inside other people's roadmaps. The thesis is narrow and the catalog is broad: production-grade, self-serve tools for the creators and craftspeople the category has under-served. One brand. Many verticals. Honest pricing. No sales motion.

Arqo is one expression of the studio — the consumer surface. This case study is about everything else. What's live, what's on deck, and the architectural bet underneath the whole thing: that a single solo operator, with AI-native tooling and fifteen years of accumulated taste, can now ship the kind of product catalog that previously required a twelve-person company.

ON THE NAME The studio is named Jamanapaar — Hindi for "the other side of the Yamuna." East Delhi, where I grew up, and the side of the river outsiders looked down on. Brooklyn, where I live now, on the other side of a different river. The studio is named for the posture.
CH. 01 — THE THESIS

single brand, many verticals.

Every tools studio I respect — Obsidian, Linear, Pieter Levels' catalog, Ghost, early Figma — shares one trait: a single brand that can carry many products, because the brand is the worldview, not the SKU.

JMNPR Labs is built the same way. The consumer sees one studio with a consistent posture: craft-forward, self-serve, PPP-priced, honest about what's shipped. The catalog underneath can span screenwriting, scheduling, design tokens, realtime infra, and standards work — and still read as one coherent thing. Because it is.

self-serve, by design.

No enterprise contracts. No sales calls. No discovery decks. Every tool is live, priced, and checkout-ready. Every asset downloadable. Every spec public. The studio's operating leverage comes from not running a sales motion — not from replacing one.

This is a deliberate constraint. A one-person studio that sells enterprise is a one-person studio that stops shipping. The moment the catalog needs sales, the catalog stops growing. I'd rather ship ten more tools than close one six-figure contract.

ai-native velocity.

Every tool in the catalog is built with Claude Code, Cursor, and v0 in agentic-dev workflows. This is not decoration. It is the reason the catalog can exist. Solo-velocity at studio-quality is a very recent possibility, and the studio is built to press on that edge rather than pretend it doesn't exist.

CH. 02 — IN MARKET

jmnpr story kit.

A showrunner-grade toolkit for writers who want to move from blank page to shootable draft without buying a $500 masterclass or subscribing to yet another Notion template farm.

Fourteen story-structure worktables. Four craft worksheets. A showrunner-grade bible, a prompt deck, a craft drills set, and a fully wired Obsidian vault. Three tiers at $29 / $79 / $149, sold through Lemon Squeezy, rendered out of a typed React-PDF pipeline that reads from a published @jmnpr/craft package.

THE REUSABLE BIT The pipeline is the reusable bit. Every future PDF-native tool in the catalog ships off the same infrastructure. Story Kit happens to be the first.

CH. 03 — ON DECK

five flagships.

These are the five products the studio is shipping next. Infra is built or mostly-built on all of them. GTM is the next step. I'm naming them here because the catalog's credibility depends on telling you what's coming, not just what's done.

01 — realtime viz starter

The infra that ran Futbolis.live at 100k concurrent users on $19/month, extracted as an npm starter. Edge caching, marker lifecycle, geocoding fallback, mobile-first architecture — packaged as a Next.js starter with a full case-study writeup.

  • For — solo devs and small teams shipping realtime dashboards without a cloud bill.
  • Status — infra live, starter extraction in progress.

02 — token lab

A Figma plugin + CLI + drift dashboard that keeps design tokens honest across Figma and code. The plugin you wish your design system had. Built after replacing Storybook in-house at a global EdTech platform and realizing the token layer was the actual bottleneck, not the component one.

  • For — design systems teams, multi-brand product orgs, the founder who's tired of token drift PRs.
  • Status — plugin live in private beta, public Q2.

03 — stripboard

A real-time scheduling app for indie 1st ADs, built on a constraint solver with cast-hold logic and simulated annealing. Movie Magic should have been replaced ten years ago. Nobody did it. Stripboard does.

  • For — indie 1st ADs, production coordinators, line producers on projects that can't justify Movie Magic's enterprise pricing.
  • Status — constraint solver shipped, UI in build.

04 — beats

A hosted beat-board with Yjs CRDT collab, presence, and first-class motion. Studio-quality real-time editor for writers' rooms and solo writers alike. Shares the collab stack with Arqo; same engine, different surface.

  • For — writers' rooms, indie teams, solo writers who want a beat-board that doesn't fight them.
  • Status — collab layer battle-tested in Arqo, standalone surface in build.

05 — .jmpr

An open container format for end-to-end filmmaking projects. Script, beats, shot list, schedule, assets, manifests — one signed, spec'd container. The format the industry has been missing and nobody's been willing to publish. Standards-layer work, given away free.

  • For — every tool in the filmmaking pipeline, starting with JMNPR's own — but open on purpose.
  • Status — spec drafted, reference implementation in build.

CH. 05 — WHAT I'M NOT BUILDING

by design.

OFF-LIMITS

  • No enterprise contracts. No SSO-gated pricing. No sales calls.
  • No AI-generates-your-script tools. JMNPR does not ship one-click generation of human creative work. Ever.
  • No subscription-by-default. Some tools are one-time. Some are subscription where it earns the model. No subscription as lock-in.
  • No studio deals that require an NDA to look at the product.
  • No competing with my own employer. EF's surface stays EF's.
CH. 06 — WHY NOW

the window is open.

Solo-velocity at studio-quality is a new thing. It's been possible for maybe eighteen months and it'll be the default in five years. The window where a single operator with taste and agentic-dev tooling can ship a tools catalog that would've required a Series A team is open right now. JMNPR Labs is built to ride that window.
